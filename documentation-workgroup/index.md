문서 도구 워크그룹은 Swift의 문서화 경험을 안내하는 운영 팀입니다.

문서 도구 워크그룹은 다음 활동을 수행합니다:

- Swift 커뮤니티의 문서화 요구를 해결하는 작업을 정의하고
  우선순위를 정합니다.
- Swift-DocC 등의 문서화 도구 개발을 적극적으로 안내합니다.
- Swift의 문서화 경험에 대한 기여를 관리하는 프로세스를
  정의합니다.
- Swift 커뮤니티의 요구에 대한 피드백을 [Ecosystem Steering Group](/ecosystem-steering-group/)에 전달합니다.

## 헌장

문서 도구 워크그룹의 주된 목표는 Swift 커뮤니티에 훌륭한 문서화 경험을
제공하는 것입니다. 이를 위해 워크그룹은 생태계와 향후 Swift 언어 문서화
작업에 제공하고자 하는 경험을 지원하는 Swift-DocC 같은 도구를 개발합니다.
시간이 지남에 따라 워크그룹은 양질의 문서 작성 지침을 정의하고
Swift 전반에 걸쳐 일관된 문서화 생태계를 이끌어 나갈 것입니다.

이를 위해 워크그룹 멤버는 풀 리퀘스트나 포럼에 게시된 아이디어를 통해
문서화 경험에 대한 변경 제안을 검토하고, 문서화 경험의 목표에 부합하는
방식으로 통합할 수 있도록 피드백을 제공합니다.

워크그룹 멤버는 문서화 UI/UX 디자인, 문서 작성 구문 등
Swift 문서화 경험의 다양한 측면을 개선하는 프로젝트를 시작할 수도 있습니다.

문서 도구 워크그룹 멤버는 Swift Core Team과 Swift 프로젝트 리드의
재량에 따라 활동하며, 프로젝트 리드가 워크그룹 결정에 대한 최종 권한을
가집니다.

## 멤버십

문서 도구 워크그룹 멤버는 위 헌장에 명시된 대로 Swift의 문서화 경험을
관리합니다. 워크그룹에 가입하고 싶으시면 포럼에서
`@swift-documentation-workgroup`으로 연락해 주세요.

3개월 연속 참여하지 않은 비활성 멤버에게는 그룹 잔류 의사를
확인합니다. 6개월간 활동이 없으면 그룹에서 제거됩니다.

Core Team은 워크그룹의 의장도 선출합니다. 의장은 워크그룹에 대한
특별한 권한은 없지만, 다음을 포함하여 원활한 운영을 책임집니다:

- 정기 회의를 조직하고 주도합니다.
- 워크그룹이 커뮤니티와 효과적으로 소통하도록 합니다.
- Core Team에 안건을 상정해야 할 때 워크그룹 대표와 Core Team 간
  회의를 조율합니다.

워크그룹 멤버는 가능한 한 합의를 통해 독립적으로 결정을 내리며,
중요한 결정에 대해 합의에 이르기 어려운 경우 Core Team에 안건을
상정합니다.

## 소통

문서 도구 워크그룹은 Swift-DocC에 대한 일반적인 논의에
[Swift-DocC 포럼](https://forums.swift.org/c/development/swift-docc)을 통해
더 넓은 Swift 커뮤니티와 소통합니다. 워크그룹에 비공개로 연락하려면
Swift 포럼에서 `@swift-documentation-workgroup`으로 메시지를 보내면 됩니다.

## 커뮤니티 참여

Swift의 문서화 경험에 기여하고 문서 도구 워크그룹의 이니셔티브에
참여하는 것을 환영합니다. 기여에 관심이 있으시면 다음을 고려해 주세요:

- 문서 도구 워크그룹의 WebEx 공개 회의에 참여합니다(포럼에서
  `@swift-documentation-workgroup`으로 연락하여 자세한 내용을 확인하세요).
- Swift 포럼에서 아이디어를 논의합니다. 예를 들어 Swift-DocC 관련 아이디어는
  [Swift-DocC 포럼](https://forums.swift.org/c/development/swift-docc)에서
  논의할 수 있습니다.
- 문서 도구 워크그룹이 관리하는 프로젝트의 개선 사항과 버그를 추적하는 이슈를 엽니다:
  [Swift-DocC](https://github.com/swiftlang/swift-docc/issues), [The Swift Programming Language 책](https://github.com/swiftlang/swift-book/issues).
- [Swift 멘토십 프로그램](/mentorship)에 참여합니다.
