Swift Server Workgroup과 Swift on Server 커뮤니티에서 서버에서 Swift를 사용하기 위한 다양한 가이드를 개발했습니다. 이 가이드는 Linux에서 Swift Server 애플리케이션을 운영하는 팀과 개인을 돕고, 이러한 개발을 시작하려는 분들에게 방향을 제시하기 위해 설계되었습니다.

이 가이드는 애플리케이션의 컴파일, 테스트, 배포 및 디버그 방법에 초점을 맞추고 해당 분야의 팁을 제공합니다.

- [빌드](/documentation/server/guides/building.html)
- [테스트](/documentation/server/guides/testing.html)
- [메모리 누수 디버깅](/documentation/server/guides/memory-leaks-and-usage.html)
- [성능 문제 해결 및 분석](/documentation/server/guides/performance.html)
- [할당 최적화](/documentation/server/guides/allocations.html)
- [멀티스레딩 이슈 및 메모리 검사 디버깅](/documentation/server/guides/llvm-sanitizers.html)
- [배포](/documentation/server/guides/deployment.html)
- [패키징](/documentation/server/guides/packaging.html)
- [Passkeys](/documentation/server/guides/passkeys.html)

추가로 라이브러리 개발자를 위한 전용 가이드도 있습니다:

- [로그 레벨](/documentation/server/guides/libraries/log-levels.html)
- [Swift Concurrency 도입](/documentation/server/guides/libraries/concurrency-adoption-guidelines.html)

_이 가이드는 커뮤니티의 노력으로 만들어졌으며, [Swift.org 사이트](https://github.com/swiftlang/swift-org-website)에 풀 리퀘스트를 제출하여 팁과 노하우를 공유하는 데 모든 분의 참여를 환영합니다_.
