이 글에서는 Swift 3.0의 목표, 릴리스 프로세스, 예상 일정을 설명합니다.

Swift 3.0은 Swift 2.2와 소스 *호환되지 않는* 메이저 릴리스입니다. 언어와 Swift 표준 라이브러리에 대한 근본적인 변경을 포함합니다. Swift 3.0의 구현된 변경 사항 전체 목록은 [Swift Evolution 사이트](https://github.com/swiftlang/swift-evolution#implemented-proposals-for-swift-3)에서 확인할 수 있습니다.

Swift 3.0은 [Swift Package Manager](/documentation/package-manager/)를 포함하는 첫 번째 릴리스이기도 합니다. Swift Package Manager는 아직 초기 개발 중이지만, 크로스 플랫폼 Swift 패키지의 개발과 배포를 지원합니다. Swift Package Manager는 Darwin과 Linux 모두에서 사용 가능합니다.

Linux에서는 Swift 3이 [Swift Core Libraries](/documentation/core-libraries/)를 포함하는 첫 번째 릴리스이기도 합니다.

Swift 3.0은 2016년 하반기에 릴리스될 예정입니다. Swift.org 릴리스 외에도 Swift 3.0은 향후 Xcode 버전에 포함될 예정입니다.

## Developer Preview

- Swift 3.0은 안정적이고 검증된 Swift 3 빌드를 제공하는 일련의 developer preview(즉 "시드" 또는 "베타")를 갖게 됩니다. 목표는 사용자가 최신 `master` 스냅샷 대신 [다운로드](/download/)하여 사용할 수 있는(그리고 버그를 보고할 수 있는) 더 안정적이고 검증된 Swift 바이너리를 제공하는 것입니다.

- Developer Preview 간의 주기는 불규칙할 수 있지만 대략 4-6주 간격입니다. `master`에 들어오는 변경의 양과 developer preview 브랜치를 안정화하는 데 필요한 시간에 따라 부분적으로 결정됩니다.

- Swift 3.0은 마지막 developer preview 브랜치에서 "GM"으로 선언됩니다.

- Developer preview에 포함되는 내용은 해당 릴리스 매니저(아래 참조)가 관리합니다.

## Swift 3.0에 변경 사항 반영

### 브랜치

- **master**: Swift 3.0 개발은 `master`에서 이루어집니다. 마지막 developer preview 브랜치가 생성될 때까지 `master`에 들어가는 모든 변경은 최종 Swift 3.0 릴리스의 일부가 됩니다. 그 시점부터 `master`는 향후 Swift 릴리스의 개발을 추적합니다.

- **swift-3.0-preview-*\<X\>*-branch**: Developer preview 브랜치는 `master`에서 생성됩니다. 이 브랜치에 대한 모든 변경은 지속적 통합 테스트를 시작하는 풀 리퀘스트를 통해 제출해야 합니다. 해당 저장소의 릴리스 매니저가 developer preview 브랜치에 풀 리퀘스트 병합을 승인합니다.

- **swift-3.0-branch**: `master`에서 생성되는 마지막 developer preview 브랜치는 `swift-3.0-branch`로 불립니다. 이것이 최종 "릴리스 브랜치"가 됩니다.

### Swift 3.0에 변경을 반영하는 원칙

- Swift 3.0이 수렴됨에 따라 릴리스의 핵심 목표에 부합하는 변경만 고려됩니다.

- 언어의 소스 호환성을 깨는 변경은 사안별로 검토됩니다.

- Swift 3.0의 모든 언어 및 API 변경은 [Swift Evolution](https://github.com/swiftlang/swift-evolution) 프로세스를 거칩니다.

- 변경 수용 기준(릴리스 매니저가 설정)은 릴리스가 수렴됨에 따라 점점 더 엄격해집니다. 동일한 정책이 본질적으로 미니 릴리스인 developer preview 브랜치에도 적용됩니다.

## 일정

- 첫 번째 developer preview 브랜치 `swift-3.0-preview-1-branch`는 **5월 12일**에 `master`에서 생성됩니다. 4-6주 후에 릴리스됩니다.

- 마지막 developer preview 브랜치 &mdash; `swift-3.0-branch` &mdash; 생성 날짜는 아직 정해지지 않았습니다. 날짜가 결정되면 [swift-dev]와 이 게시물 업데이트를 통해 공지됩니다.

## 릴리스 매니저

릴리스의 전반적인 관리는 다음 담당자들이 감독하며, Swift 3.0 릴리스가 수렴됨에 따라 변경에 대한 엄격한 관리가 적용되는 시점을 공지합니다:

- [Ted Kremenek]이 Swift 3.0의 전체 릴리스 매니저입니다.

- [Frédéric Riss](https://github.com/fredriss)가 [swift-llvm]과 [swift-clang]의 릴리스 매니저입니다.

- [Kate Stone](https://github.com/k8stone)이 [swift-lldb]의 릴리스 매니저입니다.

- [Tony Parker](https://github.com/parkera)가 [swift-corelibs-foundation]의 릴리스 매니저입니다.

- [Daniel Steffen](https://github.com/das)이 [swift-corelibs-libdispatch]의 릴리스 매니저입니다.

- Mike Ferris가 [swift-corelibs-xctest]의 릴리스 매니저입니다.

- [Rick Ballard](https://github.com/rballard)가 [swift-package-manager]의 릴리스 매니저입니다.

릴리스 관리 프로세스에 관한 질문은 [swift-dev]나 [Ted Kremenek]에게 직접 이메일하시기 바랍니다.

## Developer Preview 풀 리퀘스트

Developer preview 브랜치에 포함을 추천하는 모든 풀 리퀘스트에는 다음 정보가 포함되어야 합니다:

- **설명**: 수정하는 이슈 또는 개선 사항에 대한 설명. 간결해도 되지만 명확해야 합니다.

- **범위**: 변경의 영향/중요도 평가. 예를 들어, 소스 호환성을 깨는 언어 변경인지 등.

- **SR 이슈**: [bugs.swift.org](https://bugs.swift.org)의 이슈/개선 사항을 수정/구현하는 경우 해당 SR.

- **위험도**: 이 변경을 릴리스에 반영하는 것에 대한 (구체적인) 위험.

- **테스트**: 이 변경의 영향을 추가로 검증하기 위해 수행된 또는 필요한 구체적인 테스트.

영향을 받는 컴포넌트의 [코드 오너](/community/#code-owners) 한 명 이상이 변경을 리뷰해야 합니다. 기술 리뷰는 코드 오너가 위임하거나 적절하다고 판단되는 경우 요청할 수 있습니다.

Developer preview 브랜치에 반영되는 **모든 변경**은 해당 릴리스 매니저가 수락하는 **풀 리퀘스트를 통해야 합니다**.
