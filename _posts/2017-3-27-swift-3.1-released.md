Swift 3.1이 공식 출시되었습니다! Swift 3.1은 표준 라이브러리의 개선과 정제를 포함하는 마이너 릴리스입니다. IBM과 커뮤니티 멤버들의 노력 덕분에 Linux에서의 Swift 구현에 대한 많은 업데이트도 포함됩니다. Swift Package Manager에도 다수의 업데이트가 있습니다.

### 언어 업데이트
Swift 3.1은 마이너 언어 릴리스이며 Swift 3.0과 소스 호환됩니다. Swift [Evolution 프로세스](/contributing/#participating-in-the-swift-evolution-process)를 거친 다음 언어 변경 사항과 업데이트를 포함합니다:

#### 새로운 <code>`Sequence`</code> 프로토콜 멤버

`Sequence` 프로토콜에 두 개의 새로운 멤버가 추가되었습니다:

~~~swift
protocol Sequence {
  // ...
  /// Returns a subsequence by skipping elements while `predicate` returns
  /// `true` and returning the remainder.
  func drop(while predicate: (Self.Iterator.Element) throws -> Bool) rethrows -> Self.SubSequence
  /// Returns a subsequence containing the initial elements until `predicate`
  /// returns `false` and skipping the remainder.
  func prefix(while predicate: (Self.Iterator.Element) throws -> Bool) rethrows -> Self.SubSequence
}
~~~

자세한 내용: [SE-0045: stdlib에 `prefix(while:)`과 `drop(while:)` 추가](https://github.com/swiftlang/swift-evolution/blob/master/proposals/0045-scan-takewhile-dropwhile.md)

#### Swift 버전별 가용성
Swift 3.1은 `@availability` 속성을 확장하여 선언의 라이프사이클을 나타내는 데 Swift 버전을 사용할 수 있게 합니다. 예를 들어, Swift 3.1에서 제거된 API는 다음과 같이 작성합니다:

~~~swift
@available(swift, obsoleted: 3.1)
class Foo {
  //...
}
~~~

자세한 내용: [SE-0141: Swift 버전별 가용성](https://github.com/swiftlang/swift-evolution/blob/master/proposals/0141-available-by-swift-version.md)

#### 개선된 숫자 변환 이니셜라이저
Swift 3.1은 모든 숫자 타입에 정보 손실 없이 성공적으로 완료되거나 nil을 반환하는 새로운 변환 이니셜라이저 패밀리를 추가합니다.

자세한 내용: [SE-0080: Failable 숫자 변환 이니셜라이저](https://github.com/swiftlang/swift-evolution/blob/master/proposals/0080-failable-numeric-initializers.md)

#### UnsafeMutablePointer.initialize(from:) 지원 중단 및 대체
`Collection`을 받는 `UnsafeMutablePointer.initialize(from:)` 버전이 `Sequence`를 받는 `UnsafeMutableBufferPointer`의 새 메서드로 대체되어 지원 중단되었습니다. 이는 메모리 안전성을 개선하고 시퀀스로부터 더 빠른 메모리 초기화를 가능하게 합니다.

자세한 내용: [SE-0147: UnsafeMutablePointer.initialize(from:)을 UnsafeMutableBufferPointer로 이동](https://github.com/swiftlang/swift-evolution/blob/master/proposals/0147-move-unsafe-initialize-from.md)

#### Linux 구현 개선

* `NSDecimal` 구현
* `NSLengthFormatter` 구현
* `Progress` 구현
* API 범위 확장 및 `libdispatch` 사용 최적화를 포함한 `URLSession` 기능의 많은 개선
* `NSArray`, `NSAttributedString` 등의 API 범위 개선
* `Data`의 상당한 성능 개선. [자세한 내용은 여기를 참조](https://github.com/swiftlang/swift-corelibs-foundation/blob/master/Docs/Performance%20Refinement%20of%20Data.md)
* JSON 직렬화 성능 개선
* `NSUUID`, `NSURLComponents` 등의 메모리 누수 수정
* 특히 `URLSession`의 테스트 커버리지 개선

### Package Manager 업데이트

#### 편집 가능한 패키지

패키지 의존성은 이제 기본적으로 도구가 관리하는 빌드 디렉터리에 저장되며, 새로운 `swift package edit` 명령을 통해 사용자가 패키지를 "편집 시작"하여 사용자의 관리 하(`Packages` 디렉터리)로 이동시키고, 의존성 업데이트에서 제외하며, 해당 패키지에 대한 변경을 커밋하고 푸시할 수 있습니다.

자세한 내용: [SE-0082: Package Manager 편집 가능한 패키지](https://github.com/swiftlang/swift-evolution/blob/master/proposals/0082-swiftpm-package-edit.md)

#### 버전 고정

사용한 각 의존성의 버전이 `Package.pins` 파일에 기록되며, 이 파일을 체크인하여 패키지의 다른 사용자와 해당 버전을 공유할 수 있습니다. `swift package pin`과 `swift package unpin` 명령으로 추가 제어가 가능합니다. 의존성 해석 시 기본적으로 고정된 버전을 가져오지만, `swift package update`는 허용 가능한 최신 의존성 버전으로 다시 해석하고 핀 파일을 업데이트합니다.

자세한 내용: [SE-0145: Package Manager 버전 고정](https://github.com/swiftlang/swift-evolution/blob/master/proposals/0145-package-manager-version-pinning.md)

#### 도구 버전

패키지는 이제 필요한 Swift 도구의 최소 버전을 지정할 수 있습니다. 이 요구사항은 `swift package tools-version` 명령으로 편집하며 `Package.swift` 매니페스트 상단에 기록됩니다. 사용 중인 것보다 새로운 Swift 도구를 필요로 하는 패키지 버전은 의존성 해석에서 무시되므로, 이전 Swift 도구를 사용하는 클라이언트를 깨뜨리지 않고 새로운 Swift 기능을 채택할 수 있습니다. 필요한 최소 도구 버전은 `Package.swift` 매니페스트를 해석하는 데 사용되는 Swift 언어 버전과 사용 가능한 `PackageDescription` API 버전을 결정합니다.

자세한 내용: [SE-0152: Package Manager 도구 버전](https://github.com/swiftlang/swift-evolution/blob/master/proposals/0152-package-manager-tools-version.md)

#### Swift 언어 호환성 버전

패키지는 이제 소스가 Swift 3 또는 Swift 4 언어 버전으로 작성되었는지 지정할 수 있습니다. 지정하지 않으면 패키지의 최소 Swift 도구 버전에서 기본값이 추론됩니다.

자세한 내용: [SE-0151: Package Manager Swift 언어 호환성 버전](https://github.com/swiftlang/swift-evolution/blob/master/proposals/0151-package-manager-swift-language-compatibility-version.md)

#### 기타 Package Manager 개선

* 이전에 잘못된 의존성 버전으로 해석될 수 있었던 경우에 대해 패키지 의존성 해석이 올바르게 수정되었습니다. 의존성 순환이 이제 빌드 중에 감지되며, 증분 빌드가 가능한 경우 더 적은 소스를 다시 빌드합니다.

* `swift test`가 이제 `--parallel` 플래그로 병렬 테스트 실행을 지원합니다. `swift build`, `swift test`, 의존성을 해석하는 모든 `swift package` 명령은 이제 `--enable-prefetching` 플래그로 의존성의 병렬 가져오기를 지원합니다.

Swift Package Manager 문서는 [저장소](https://github.com/swiftlang/swift-package-manager/tree/swift-3.1-branch/Documentation)에서 확인할 수 있습니다.


### Swift 3.1로 마이그레이션
Swift 3.1은 Swift 3.0과 소스 호환됩니다. 이전 Swift 릴리스에서 Swift 3.1로 전환하는 데 도움이 되도록 [Xcode 8.3]에는 필요한 많은 소스 변경을 자동으로 처리하는 코드 마이그레이터가 포함되어 있습니다. 또한 많은 변경 사항을 안내하는 [마이그레이션 가이드](/migration-guide/)도 제공됩니다 — 특히 기계적이지 않고 더 직접적인 검토가 필요한 부분에 대해서 말입니다.
