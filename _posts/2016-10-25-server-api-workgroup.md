Swift가 Linux에서 사용 가능해진 이후 서버에서 Swift를 사용하는 것에 대한 관심이 폭발적으로 증가했으며, Kitura, Vapor, Perfect, Zewo를 비롯한 수많은 웹 프레임워크가 등장했습니다. Swift 생태계의 중요한 부분이자 적극적으로 육성하고자 하는 분야로서, 오늘 Server APIs 워크 그룹의 결성을 발표합니다.

이 워크 그룹은 서버 애플리케이션과 프레임워크 구축에 관심 있는 커뮤니티 참여자들이 함께 모여 새로운 Swift API를 제공하는 프레임워크를 마련합니다. 이 API는 서버 측 기능 개발의 기본 구성 요소로서 저수준 "서버" 함수를 제공하며, 이런 기능에 대해 일반적으로 플랫폼별 C 라이브러리에 의존하는 것을 제거합니다. 이를 통해 시스템 프로그래밍 기술과 여러 플랫폼에 대한 지식 없이도 더 많은 개발자가 순수 Swift 코드로 프레임워크와 서버 애플리케이션을 만들 수 있게 됩니다.

워크 그룹은 처음에 네트워킹, 보안, HTTP/WebSocket 파싱을 위한 API를 검토하며, 누구나 간단하고 안전한 HTTP 서버를 구축하거나 pub/sub 메시지 브로커 같은 서버 프레임워크를 만들 수 있도록 하는 것을 목표로 합니다.

자세한 내용은 [Server APIs 프로젝트 페이지](/sswg/)를 참조하세요.

> 편집자 주: Server APIs Workgroup은 Swift Server Workgroup으로 이름이 변경되었습니다. 이 글의 링크는 새 URL로 업데이트되었습니다.
