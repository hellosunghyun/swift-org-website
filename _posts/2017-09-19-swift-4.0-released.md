Swift 4가 공식 출시되었습니다! Swift 4는 Swift 3의 강점을 기반으로 더 높은 견고성과 안정성을 제공하고, Swift 3과의 소스 코드 호환성을 유지하며, 표준 라이브러리를 개선하고, 아카이벌 및 직렬화 같은 기능을 추가합니다.

간략한 개요는 [WWDC 2017: What's New in Swift](https://developer.apple.com/videos/play/wwdc2017/402/) 프레젠테이션에서 확인하고, Ole Begemann이 만든 이 [playground](https://github.com/ole/whats-new-in-swift-4) 에서 새로운 기능을 직접 사용해 볼 수 있습니다.

### 언어 업데이트

Swift 4.0은 메이저 언어 릴리스이며 Swift Evolution 프로세스를 거친 다음 언어 변경 및 업데이트를 포함합니다:

#### String

Swift 4는 유니코드 정확성을 유지하면서 더 빠르고 사용하기 쉬운 `String` 구현을 포함하며, 서브스트링의 생성, 사용, 관리 지원을 추가합니다.

Swift 4는 Collection 타입의 생성, 사용, 관리에 대한 개선을 추가합니다.

#### 아카이벌과 직렬화

Swift 4는 struct와 enum 타입의 아카이벌을 지원하고 JSON, plist 같은 외부 포맷으로의 타입 안전 직렬화를 가능하게 합니다.

자세한 내용: [SE-0166 Swift Archival & Serialization](https://github.com/swiftlang/swift-evolution/blob/master/proposals/0166-swift-archival-serialization.md)

#### 새로운 호환 모드

Swift 4에서는 새 버전의 컴파일러를 사용하기 위해 코드를 수정할 필요가 없을 수 있습니다. 컴파일러는 두 가지 언어 모드를 지원합니다:

* *Swift 3.2*: 이 모드에서 컴파일러는 Swift 3.x 컴파일러로 빌드된 대부분의 소스를 수용합니다. 이 수준의 소스 호환성을 제공하기 위해 기존 API(표준 라이브러리의 일부이거나 Apple이 제공하는 API)에 대한 업데이트는 이 모드에 나타나지 않습니다. Swift 4의 대부분의 새로운 언어 기능은 이 언어 모드에서 사용 가능합니다.

* *Swift 4.0*: 이 모드는 모든 Swift 4.0 언어 및 API 변경을 포함합니다. 많은 프로젝트에서 일부 소스 마이그레이션이 필요하지만, 이전 Swift 릴리스 간 주요 변경에 비해 소스 변경 수는 상당히 적습니다.

언어 모드는 `-swift-version` 플래그로 컴파일러에 지정되며, Swift Package Manager와 Xcode가 자동으로 처리합니다.

이런 언어 모드의 장점은 새로운 Swift 4 컴파일러를 사용하기 시작하면서 자신만의 속도로, 모듈 단위로 Swift 4의 새 기능을 활용하며 점진적으로 마이그레이션할 수 있다는 것입니다.

Swift 4 마이그레이션과 호환 모드에 대한 자세한 내용은 다음을 참조하세요: [Migrating to Swift 4](/migration-guide-swift4/)

### Package Manager 업데이트

Swift 4는 Swift Package Manager에 새로운 워크플로우 기능과 더 완전한 API를 도입합니다:

* It’s now easier to develop multiple packages in tandem before tagging your first official release, or to work on a branch of multiple packages together.
* 패키지 제품이 공식화되어 패키지가 클라이언트에 공개하는 라이브러리를 제어할 수 있게 되었습니다.
* 새로운 Package API는 패키지가 다양한 새 설정을 지정할 수 있게 하여, 패키지 작성자에게 패키지 빌드 방식이나 디스크의 소스 구성에 대한 더 많은 제어 권한을 부여합니다. 전반적으로 패키지를 만드는 데 사용되는 API가 더 깔끔하고 명확해졌으며, 이전 패키지와의 소스 호환성은 유지됩니다.
* macOS에서 Swift 패키지 빌드는 이제 네트워크 접근과 파일 시스템 수정을 방지하는 샌드박스에서 수행되어, 악의적으로 조작된 매니페스트의 영향을 완화합니다.

또한 Swift Package Manager는 Swift 3.1에서 도입된 package manager tools versioning을 기반으로 합니다 ([SE-0159](https://github.com/swiftlang/swift-evolution/blob/master/proposals/0152-package-manager-tools-version.md))  — 패키지 빌드에 필요한 Swift 버전을 지정할 수 있으며, 이제 Swift 4도 포함됩니다.
