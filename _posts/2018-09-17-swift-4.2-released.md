Swift 4.2가 공식 출시되었습니다! Swift 4.2는 Swift 4의 강점을 기반으로 더 빠른 컴파일 시간, 개선된 디버깅 경험, 표준 라이브러리 업데이트, 바이너리 호환성 수렴 등을 제공합니다.

[WWDC 2018: What's New in Swift](https://developer.apple.com/videos/play/wwdc2018/401/) 프레젠테이션을 시청하여 간략한 개요를 확인하고, Ole Begemann이 만든 이 [playground](https://oleb.net/blog/2018/06/whats-new-in-swift-4-2-playground/)에서 새로운 기능을 직접 체험해 볼 수 있습니다.

### 언어 업데이트

Swift 4.2는 주요 언어 릴리스로, Swift Evolution 프로세스를 거친 다음과 같은 언어 변경 및 업데이트가 포함되어 있습니다:

#### 제네릭 개선

조건부 적합성 지원이 완료됨에 따라 Swift 4.2는 언어에서 제네릭 비전을 구현하는 데 상당한 진전을 이루었습니다. 제네릭에 대한 더 나은 지원은 궁극적으로 코드에서 필요한 보일러플레이트를 줄이고 더 많은 코드를 재사용할 수 있게 합니다. 조건부 적합성에 대한 자세한 내용은 다음을 참고하세요:

* [SE-0143 조건부 적합성](https://github.com/swiftlang/swift-evolution/blob/master/proposals/0143-conditional-conformances.md)

더 자세한 내용은 WWDC 2018의 [Swift Generics](https://developer.apple.com/videos/play/wwdc2018/406/) 프레젠테이션을 참고하세요.


#### 표준 라이브러리 업데이트

Swift 4.2의 표준 라이브러리에는 Hashable 프로토콜 개선과 새로운 통합 무작위화 함수 및 프로토콜을 포함한 여러 새 기능이 포함되어 있습니다.

자세한 내용은 다음을 참고하세요:

* [SE-0197 표준 라이브러리에 in-place removeAll(where:) 추가](https://github.com/swiftlang/swift-evolution/blob/master/proposals/0197-remove-where.md)
* [SE-0199 Bool에 toggle 추가](https://github.com/swiftlang/swift-evolution/blob/master/proposals/0199-bool-toggle.md)
* [SE-0202 Random 통합](https://github.com/swiftlang/swift-evolution/blob/master/proposals/0202-random-unification.md)
* [SE-0204 last(where:) 및 lastIndex(where:) 메서드 추가](https://github.com/swiftlang/swift-evolution/blob/master/proposals/0204-add-last-methods.md)
* [SE-0206 Hashable 개선](https://github.com/swiftlang/swift-evolution/blob/master/proposals/0206-hashable-enhancements.md)
* [SE-0207 Sequence에 allSatisfy 알고리즘 추가](https://github.com/swiftlang/swift-evolution/blob/master/proposals/0207-containsOnly.md)

#### 바이너리 호환성

Swift 4.2는 언어를 안정화된 ABI에 더 가깝게 만들어, 향후 Swift 릴리스를 위한 바이너리 호환성을 제공합니다. 이 작업의 현재 진행 상황에 대한 자세한 내용은 [ABI 대시보드](/abi-stability/)를 참고하세요.

#### 추가 언어 및 컴파일러 업데이트

Swift 4.2에는 개발자 경험에 눈에 띄는 영향을 미치는 여러 개선 사항이 포함되어 있습니다:

* 배치 모드 컴파일 지원으로 빌드 시간 단축
* retain/release 사이클의 호출 규약 변경으로 코드 크기 감소 및 런타임 성능 향상
* -Onone에서의 지연 SIL 역직렬화
* 재귀 메타데이터 지원
* 더 컴팩트한 리플렉션 메타데이터
* 호출 지점에서 기본 인자 인라이닝
* init declname, 블록당 여러 case 레이블이 있는 switch, 제네릭 이니셜라이저 상속 등 오래된 버그 수정

Swift 4.2는 Swift Evolution 프로세스의 다음 언어 제안도 구현합니다:

* [SE-0054 ImplicitlyUnwrappedOptional 타입 폐지](https://github.com/swiftlang/swift-evolution/blob/master/proposals/0054-abolish-iuo.md). 재구현에 대한 자세한 내용은 [여기](/blog/iuo/)를 참고하세요.
* [SE-0079 옵셔널 바인딩을 사용하여 약한 참조에서 강한 참조로 self 승격 허용](https://github.com/swiftlang/swift-evolution/blob/master/proposals/0079-upgrade-self-from-weak-to-strong.md)
* [SE-0193 크로스 모듈 인라이닝 및 특수화](https://github.com/swiftlang/swift-evolution/blob/master/proposals/0193-cross-module-inlining-and-specialization.md)
* [SE-0194 열거형 케이스의 파생 컬렉션](https://github.com/swiftlang/swift-evolution/blob/master/proposals/0194-derived-collection-of-enum-cases.md)
* [SE-0195 사용자 정의 "Dynamic Member Lookup" 타입 도입](https://github.com/swiftlang/swift-evolution/blob/master/proposals/0195-dynamic-member-lookup.md)
* [SE-0196 컴파일러 진단 디렉티브](https://github.com/swiftlang/swift-evolution/blob/master/proposals/0196-diagnostic-directives.md)
* [SE-0205 불변 값에 대한 withUnsafePointer(to:\_:) 및 withUnsafeBytes(of:\_:)](https://github.com/swiftlang/swift-evolution/blob/master/proposals/0205-withUnsafePointer-for-lets.md)
* [SE-0210 MemoryLayout에 offset(of:) 메서드 추가](https://github.com/swiftlang/swift-evolution/blob/master/proposals/0210-key-path-offset.md)
* [SE-0212 컴파일러 버전 디렉티브](https://github.com/swiftlang/swift-evolution/blob/master/proposals/0212-compiler-version-directive.md)


### Package Manager 업데이트

Swift 4.2에서 Swift Package Manager에 새로운 기능이 도입되었습니다:

* **배치 모드 지원.** Swift 타겟이 이제 Swift 컴파일러의 배치 모드를 사용하여 컴파일됩니다.

* **자동 Xcode 프로젝트 생성.** generate-xcodeproj에 파일 시스템을 감시하고 필요할 때 Xcode 프로젝트를 자동으로 재생성하는 새로운 --watch 옵션이 추가되었습니다. 이 기능은 watchman 도구를 사용하여 필요한 파일을 감시합니다.

* **개선된 스킴 생성 로직.** 스킴 생성 로직이 개선되어 다음과 같이 스킴을 생성합니다:
  * 루트 패키지의 모든 일반 타겟과 테스트 타겟을 포함하는 하나의 스킴.
  * 실행 가능 타겟당 하나의 스킴으로, 실행 가능 타겟의 의존성과 교차하는 의존성을 가진 테스트 타겟을 포함합니다.

Swift 4.2는 Swift Evolution 프로세스의 다음 Package Manager 제안도 구현합니다:

* [SE-0201 Package Manager 로컬 의존성](https://github.com/swiftlang/swift-evolution/blob/master/proposals/0201-package-manager-local-dependencies.md). SwiftPM이 이제 git URL 대신 디스크 경로를 사용하여 패키지에 대한 의존성을 선언할 수 있습니다. 이를 위해 패키지의 도구 버전을 4.2로 업데이트해야 합니다.
* [SE-0208 Package Manager 시스템 라이브러리 타겟](https://github.com/swiftlang/swift-evolution/blob/master/proposals/0208-package-manager-system-library-targets.md). 도구 버전 4.2의 PackageDescription API가 새로운 유형의 타겟인 "시스템 라이브러리 타겟"을 지원하여, 현재의 시스템 모듈 패키지 기능을 패키지 수준에서 타겟 수준으로 이동합니다.
* [SE-0209 Package Manager Swift 언어 버전 API 업데이트](https://github.com/swiftlang/swift-evolution/blob/master/proposals/0209-package-manager-swift-lang-version-update.md). 도구 버전 4.2용 PackageDescription 매니페스트 API의 `swiftLanguageVersions` 프로퍼티가 정수 배열에서 SwiftVersion 열거형 배열로 변경되었습니다.

### Swift 4.2로 마이그레이션

Swift 4.2는 Swift 4, Swift 4.1, Swift 3과 소스 호환됩니다. Swift 4.2는 Swift 3과의 소스 호환성을 제공하는 마지막 릴리스입니다.

이전 Swift 릴리스에서 Swift 4.2로 전환하는 것을 돕기 위해 Apple의 Xcode 10에는 필요한 소스 변경 사항을 자동으로 처리할 수 있는 코드 마이그레이터가 포함되어 있습니다. 또한 변경 사항을 안내하는 [마이그레이션 가이드](/migration-guide-swift4.2)가 제공됩니다 — 특히 기계적이지 않고 직접적인 검토가 필요한 변경 사항에 유용합니다.
